# 1장 컴퓨터 구조

## 1. 컴퓨터 구조를 알아야 하는 이유

컴퓨터 구조를 알면 다양한 방면에서 문제를 확인하고 해결할 수 있음.
개발할 때도 단순히 코드가 작동하는 것에서 끝나는 것이 아니라 성능, 용량, 비용 측면을 고려하면서 더 효율적으로 개발할 수 있음.

---

## 2. 컴퓨터가 이해하는 정보

컴퓨터는 0과 1로 표현된 정보만 이해할 수 있음.
컴퓨터가 이해하는 정보는 데이터,명령어2가지로 나뉨.

데이터: 숫자, 문자, 사진, 영상 등과 같은 정적인 정보
명령어: 데이터를 다루고 컴퓨터를 작동시키는 정보


---

## 3. 컴퓨터의 4가지 핵심 부품

컴퓨터에는 4가지 핵심 부품이 있음.

중앙처리장치(CPU)
주기억장치(메모리)
보조기억장치 **입출력장치**

### CPU

CPU는 컴퓨터의 핵심 부품으로, ALU, 레지스터, 제어장치로 구성됨.

ALU: 컴퓨터 내부의 계산과 논리 연산을 담당함.
레지스터: 프로그램 실행 시 필요한 정보를 CPU 내부에 임시로 저장하는 역할을 함.
제어장치: 명령어를 해석하고, 필요한 정보를 가져오거나 저장할 때 제어 신호를 보내 컴퓨터의 각 부품이 올바르게 작동하도록 제어함.

### 주기억장치(메모리)

현재 실행 중인 프로그램의 명령어와 데이터를 저장하는 장치임.
CPU가 프로그램을 실행하기 위해서는 필요한 명령어와 데이터가 메모리에 저장되어 있어야 함.

### 보조기억장치

메모리는 전원이 꺼지면 저장된 정보가 사라지기 때문에 이를 보조하기 위해 사용됨.
보조기억장치는 전원이 꺼져도 데이터를 저장할 수 있으며, 대표적으로 SSD와 HDD가 있음.

### 입출력장치

컴퓨터 외부와 내부가 정보를 주고받을 수 있도록 하는 장치임.
키보드, 마우스, 모니터, 프린터기 등이 있음.

---

## 4. 메인보드와 시스템 버스

컴퓨터의 4가지 핵심 부품들은 메인보드에 연결되어 있으며, 시스템 버스를 이용하여 서로 정보를 주고받음.

시스템 버스는 크게 3가지로 나뉨.

주소 버스: 데이터가 저장되어 있거나 이동할 주소 정보를 전달함.
데이터 버스: 실제 데이터와 명령어를 전달함.
제어 버스: 읽기, 쓰기 등 컴퓨터 부품을 제어하기 위한 제어 신호를 전달함.
