# Chapter 02 데이터

컴퓨터는 0과 1만으로 이해함

---

### 정보단위
- 비트
  - 0과 1을 나타내는 가장 작은 정보단위
- 바이트
  - 8개의 비트를 묶은 단위
  - 1바이트 = 8비트
- 킬로바이트
  - 1바이트를 1000개 묶은 단위
- 메가바이트
  - 1킬로바이트를 1000개 묶은 단위
- 기가바이트
  - 1메가바이트를 1000개 묶은 단위
- 테라바이트
  - 1기가바이트를 1000개 묶은 단위

---

### 워드
- CPU가 한 번에 처리할 수 있는 데이터 크기
  - 하프워드
    - 워드의 절반 크기
  - 풀워드
    - 워드의 1배 크기
  - 더블워드
    - 워드의 2배 크기

---

### 이진법
- 0과 1만으로 모든 숫자를 표현하는 방법
- 숫자가 1을 넘어가는 시점에 자리 올림 → 이진수
- 이진수의 음수표현
  - 2의 보수를 구해서 표현

### 십진법
- 열 개의 숫자만으로 모든 수를 표현하는 방법
- 숫자가 9를 넘어가는 시점에 자리 올림 → 십진수

### 십육진법
- 수가 15를 넘어가는 시점에 자리 올림을 하는 숫자 표현 방식

### 플래그(부가정보)
- 컴퓨터 내부에서 어떤 수를 다룰 때, 이 수가 양수인지 음수인지 구분하기 위한 방법

---

### 문자집합
- 문자집합
  - 컴퓨터가 인식하고 표현할 수 있는 문자들의 모음
- 문자 인코딩
  - 사람의 문자를 컴퓨터의 언어로 변환하는 과정
- 문자 디코딩
  - 컴퓨터의 언어를 사람의 문자로 변환하는 과정

### 확장 아스키
- 다양한 문자 표현을 위해 아스키 코드에 1비트를 추가한 8비트 코드

### 한글 인코딩
- 완성형 인코딩
  - EUC-KR
    - 대표적인 인코딩 방식
- 조합형 인코딩

---

### 아스키 코드
- 아스키 문자에 대응된 고유한 수
- 0부터 127까지 총 128개 존재

---

### 유니코드
- 여러 나라의 문자를 광범위하게 표현할 수 있는 통일된 문자 집합
- UTF-8
  - 가장 대중적인, 유니코드 문자에 부여된 값을 인코딩하는 방식

---

# 느낀점

이번 챕터는 헷갈리고 어려웠던 챕터인 것 같다. 바이트에서 메가바이트, 테라바이트로 넘어가는 계산이 쉽지 않았고, 
유니코드는 외울 것도 많고 변환 과정이 헷갈려서 AI와 같이 공부했다. 
이 파트는 다뤄본 경험이 부족해서 어려운 것 같은데, 문제를 풀다 보면 좀 더 쉬워질 것 같다.
